Analysis of the Cryptosystem/code

1. Key Generation

The script sets up an RSA-like cryptosystem. Learn more about RSA from here.

  • p & q: These are two 512-bit primes. A crucial detail is the condition p % 4 == 3. This means the primes must leave a remainder of 3 when divided by 4. Apparently these kind of numbers are special in cryptography and are known as Blum Primes. They make calculating modular square roots easy using the formula x^((p+1)/4) mod p.

  • n: This is the public modulus, the 1024-bit product of p and q. The security of the system relies on the fact that factoring n back into p and q is computationally infeasible. This is because even though n is extremely large, we can get it easily by multiplying p and q. But finding two prime factors (p and q) which when multiplying result to n is very hard because the best we can do is try all combinations of numbers which is not feasible for a number that is 1024 bits which is sth like 10 raised to power 300.

  • e: This is the standard public exponent 65537.

The Public Key is (n, e) but we only know e, and the Private Key is (p, q).

2. The Goal: Forging a Signature

The final part of the code presents the challenge:

The server gives us a random message(number) m and asks for a signature s. The signature is valid if pow(s,e,n)=m which mathematically equals to s^e ≡ m (mod n). This is the standard verification process for an RSA signature. To create s, we need to calculate the e-th root of m mod n. This is impossible without the private key, which requires factoring n.

3. The Vulnerability: The Square Root Oracle

The most important part of the code is the function f(x) which is provided to us as a service (maybe Saas refers to sq. root as service or sth like that)

This complex-looking function is a Square Root Oracle. Given an input l, it calculates one of its four modular square roots mod n . It does this by:

  1. Calculating a square root modulo p (using the Blum Prime trick).
  2. Calculating a square root modulo q.
  3. Combining them using the Chinese Remainder Theorem (CRT) to get a square root modulo n.

The choice([-1, 1]) ensures it randomly picks one of the four possible roots.

The Attack Plan

The entire attack focuses on abusing the sq. root service.

Stage 1: Discovering the Modulus n

The oracle guarantees that if we send it l, it will return a y such that y² ≡ l (mod n). This means y² - l is a multiple of n.

We can exploit this to find n:

  1. Ask the oracle for the square root of 1. Let the result be y1. We now know that M1 = y1² - 1 is a multiple of n.
  2. Ask the oracle for the square root of 4. Let the result be y2. We now know that M2 = y2² - 4 is also a multiple of n.
  3. Since M1 and M2 are both multiples of n, their Greatest Common Divisor (GCD) will be n itself (or a large multiple of it, but in practice, it’s n).

With this, we can recover the full modulus n.

Stage 2: Factoring n

Now that we have n, we can use the oracle in a different way to factor it. This is a classic cryptographic attack.

  1. Choose a random number x and compute its square l = x² % n. We now know one square root of l (it’s x!).
  2. Send l to the oracle and get its result, y.
  3. We now have two numbers, x and y, where x² ≡ y² (mod n). This means x² - y² ≡ 0 (mod n), which can be factored as (x - y)(x + y) ≡ 0 (mod n).
  4. There’s a 50% chance the oracle will return a “non-trivial” root, meaning y is not equal to x or -x (mod n). In this case, the prime factors p and q are split between the terms (x - y) and (x + y).
  5. This means that gcd(x - y, n) will reveal one of the prime factors, p. The other is simply q = n // p.

We can loop this process until we find a non-trivial root, which successfully factors n.

The Exploit

With p and q found, the rest is standard RSA. You can learn about it from here

  1. Calculate phi(n) = (p - 1) * (q - 1).
  2. Calculate the private exponent d = pow(e, -1, phi_n).
  3. Receive the message m from the server.
  4. Forge the signature s = pow(m, d, n).
  5. Send s to the server to get the flag.
