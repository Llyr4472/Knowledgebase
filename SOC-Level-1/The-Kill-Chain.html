    The Kill Chain

    Jan 24, 20253 min read

    The Cyber Kill Chain framework is designed for identification and prevention of the network intrusions. The term kill chain is used in millitary to refer to the steps taken by an adversary to reach its ultimate goal.

    A typical cyber kill chain looks like:

    The Cyber Kill Chain

    1. Reconnaissance

    Reconnaissance is the first step and it provides adversary with all the info needed for getting into a system.

    2. Weaponization

    In this step, the threat actor combines malware and exploit into a deliverable payload. Most attackers usually use automated tools to generate the malware or refer to the Dark Web to purchase the malware.

    3. Delivery

    The delivery of the Weapon can be performed through plenty of ways:

    1. Phishing emails

    2. Distributing infected devices in public places

    3. Watering Hole attack

    4. Exploitation

    After gaining access to the system, the malicious actor could exploit software, system, or server-based vulnerabilities to escalate the privileges or move laterally through the network.

    Some examples of exploitation are:

    1. Zero day exploit

    2. Exploit software, hardware or even human exploit

    3. Server based exploit

    5. Installation

    Once the system is exploited, the attacker will install a persistent Backdoor in the system. This persistence can be achieved through various methods:

    1. Installing a Web Shell

    A web shell, because of its simplicity, can be well hidden and difficult to get rid of. Hence, it serves as an excellent Backdoor for an attacker.

    2. Windows Services

    Windows service run in background and can be modified to execute a malicious script or payloads. Adversary may create, or modify a windows service to execute payloads under a service name related to the OS or legitimate software.

    3. Run keys

    Adding the entry to the “run keys” for the malicious payload in the Registry or the Startup Folder. By doing that, the payload will execute each time the user logs in on the computer.

    4. Installing a Backdoor

    Adversary may install other backdoors such as Meterpreter’s backdoor, to later gain access to the system.

    In this phase, the attacker can also use the Timestomping and Masquerading techniques to avoid detection by the forensic investigator and hide the malware.

    6. Command and Control (C2)

    Now the system has been compromised and a Backdoor has been installed. But the attacker stills needs a way to communicate with the compromised system and that also without being detected. Here comes the role of C2 Server. With C2 server the attacker can easily control the host.

    7. Exfiltration

    Now the adversary finally execute their objective which may be stealing data, destroying data, continuing lateral movement, etc.
