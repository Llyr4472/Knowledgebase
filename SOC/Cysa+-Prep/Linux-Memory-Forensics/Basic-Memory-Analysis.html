Basic Memory Analysis

Basic Memory Analysis

Sep 12, 20259 min read

Memory Forensics Fundamentals is an essential aspect of digital forensics, providing critical information for understanding potential security breaches or system failures.  Analyzing the extracted memory dump file is a fundamental part of the forensics process and is performed with advanced tools such as Volatility.

Before moving on to the memory dump analysis process, let’s look at the plug-ins that can be used for Linux memory forensics with the Volatility 3 tool:

python3 vol.py --help | grep -i linux.

To get detailed information about a plugin, look at the output of the help command for that plugin.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.sockstat --help

Memory Dump Analysis

A basic “Linux memory dump analysis process” should consist of the following steps:

  • Image Identification

  • Processes and Threads

  • Network Connections

  • Linux Bash History / User Activities

  • File Analysis

  • Malware Analysis

Image Identification

First, verify the type and format of the memory dump. Choosing the right profile affects the reliability of the results, so this is a critical step for the accuracy of the analysis. Factors such as operating system version, architecture, and kernel version must be correctly identified.

It is also necessary to take the hash information of the memory dump and check the hash on the system we are going to analyze, because usually the memory capacities of the systems are large (8-16-24-32 GB). You should take into account the possibility of corruption when transferring a file of this size between systems.

Using commands that use md5sum or other hashing algorithms, you can quickly hash the memory dump file.

md5sum /var/dumps/ubuntu-memdump.lime

Processes and Threads

Next, obtain a comprehensive list of processes and threads running on the system. This can help identify suspicious or malicious processes and distinguish between normal and abnormal activity on the system. In addition, the process tree and parent-child process relationships can provide important clues for tracking down malware.

The “linux.psaux”, “linux.pstree”, and “linux.psscan” plugins included in Volatility 3 allow you to view and analyze the processes running on the memory dump file. The details of the information in this output are as follows:

PID: Process Identifier is a number that uniquely identifies each process.

PPID: Parent Process Identifier refers to the identifier of the parent process of a process that was started.

COMM: The Command Name is the name of the program that the process is running.

ARGS: Arguments is the set of command line arguments used in the process startup.

Based on this information, the commands executed on the system are analyzed one by one, and if there are abnormal or suspicious commands, the forensics process proceeds for those commands.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.psaux

The details of the “linux.psscan” plugin output are as follows:

OFFSET (P): Specifies the position (offset) of the process in memory in the physical address space.

PID: Process Identifier is a number that uniquely identifies each process.

TID: Thread Identifier identifies each thread within the process.

PPID: Parent Process Identifier is the identifier of the parent process from which a process was created.

COMM: Command Name is the name of the program that the process is executing.

EXIT_STATE: Indicates the exit state of the process, i.e. whether the process has been killed or is still running.

The details of the most important EXIT_STATES are as follows:

TASK_RUNNING: The process is actively running and does not specify an exit state.

EXIT_DEAD: The process has ended and is no longer running. This indicates that the process has ended and its resources have been reclaimed by the operating system.

EXIT_ZOMBIE: The process has ended, but is still listed in the operating system logs because an exit code has not yet been received from the parent process.

EXIT_TRACE: The process has been stopped for monitoring or debugging purposes and is waiting for an exit state.

TASK_STOPPED: The process was stopped by a signal and is no longer running.

TASK_TRACED: The process is being traced by a debugging tool.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.psscan.PsScan

Network Connections

Examine active and past network connections to understand which external servers the system is communicating with. This analysis is critical for identifying potential data leaks or connections to command and control (C&C) servers. Open ports and network traffic can help identify external attacks or remote access attempts.

The “linux.sockstat” plugin included with Volatility 3 allows you to view and analyze currently active network accesses on the memory dump file. The information in the details of this output is as follows:

NetNS: Network Namespace Number, refers to the network namespaces used for isolation, routing and management in Linux.

Pid: Process Identifier, a number used to uniquely identify each process.

FD: File Descriptor, refers to a process file descriptor associated with a socket or other file type.

Sock Offset: Specifies the physical location (offset) of the socket structure in memory.

Family: Specifies the socket family (for example, AF_INET is used for IPv4 Internet protocols).

Type: Specifies the type of socket (for example, STREAM is used for connection-oriented communication such as TCP).

Proto: Specifies the protocol used (for example, TCP).

Source Addr: Indicates the local source address, i.e. the local IP address to which the socket sends or receives data.

Source Port: Indicates the local source port, i.e. the local port number to which the socket sends or receives data.

Destination Addr: Specifies the remote destination address, i.e. the destination IP address to which the socket sends or receives data.

Destination Port: Specifies the remote destination port, i.e. the destination port number to which the socket sends or receives data.

State: Indicates the current state of the socket (e.g. ESTABLISHED indicates that the connection is established, LISTEN indicates that the socket is listening).

FilterRAW: Indicates the raw data filtering status (if any), usually including information about firewall or packet filtering rules. This column indicates whether or not a particular filter is applied.

In this example, only the connections of processes with PID numbers 829 and 907 are filtered and output.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.sockstat --pids 829 907

Linux Bash History / User Activities

In this step, examine the command line activity performed on the system. The bash history is an important resource for detecting malicious commands, suspicious activity, or changes to the system because it reveals what commands users run and what types of operations they perform on the system.

With the “linux.bash” plugin included in Volatility 3, you can view and analyze the linux bash history on the memory dump file.  The details of this output are as follows:

PID: Process Identifier refers to the unique identifier number of the process being analyzed.

Process: This is the name of the process with a particular PID.

CommandTime: This is the date and time when the command was executed.

Command: Displays the command that was executed.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.bash

File Analysis

Analyze file and directory structures on the system. Files left behind by malware, temporary files, or files that have been deleted but still exist in memory can be critical to understanding how a security incident unfolded.

With the “linux.lsof” plugin included in Volatility 3, you can see and analyze which process is associated with which file on the memory dump file.

In this example, only the file activity of the nginx process with PID 838 was filtered and output. The details of this output are as follows:

PID: Process Identifier is the unique number of the analyzed process.

Process: This is the name of the process. That is, the name of the process with a particular PID.

FD: File Descriptor is a number that identifies a specific file or socket used by the process.

Path: This is the path in the system to the file or resource that the file descriptor (FD) points to.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.lsof --pid 838

Malware Analysis

The next step is to look for malware artifacts in the memory dump. Malware often uses sophisticated techniques to disguise itself. You should perform a detailed scan for signatures, hidden processes, or malicious installation mechanisms. The presence of rootkits, backdoors, and other malware is a serious threat to system security.

The “linux.vmayarascan” plugin included in Volatility 3 is an invaluable plugin for detecting potential threats or important information on a memory dump file.

Uses include:

Malware Analysis: It can be used to detect malware samples, hidden processes, or rootkits from memory.

Pattern Search: It can be used to find specific patterns or strings (such as sensitive information or encryption keys) in system memory.

As shown in the command image below, specific strings were searched in process 838 PID, and the output in HEX form was decoded with the “xxd” tool and displayed as a string.

python3 vol.py -f /var/dumps/ubuntu-memdump.lime linux.vmayarascan --pid 838 --yara-file /root/tools/yara-rules/custom/string_search.yar

echo “6572726f72” | xxd -r -p

Thanks to the custom rule “string_search.yar”, the expressions “error” and “access” were searched as strings in the analyzed memory dump.

cat ../yara-rules/custom/string_search.yar

Drawing Conclusions and Writing a Report

Finally, all information gained from the analysis should be reported in an understandable and actionable format. This report should include the timeline of the incident, systems affected, threats identified, and recommended actions to be taken. Effective reporting is a key resource for further incident response and potential legal proceedings.

Conclusion

Memory dump analysis is performed using tools such as Volatility 3 and reveals potential threats, anomalous behavior, and user activity on the system. The basic Linux memory dump analysis process includes examining processes, network connections, user activity, file analysis, and malware analysis. Each step provides a detailed examination of activity on the system, helping forensic experts uncover the truth behind the incidents. The results are reported in detail to help determine the nature of the incident and the security-related issues.
